自动加载

尽管之前已经写过一个自动加载类的方法, 在这里重点复习和深入理解自动加载原理和相关函数的用途

分类

  • __autooad()

    • 优点: 简单易用
    • 缺点: 大型项目中由于命名规则不一致是, 该方法将会十分臃肿并可能影响效率

  • spl方法

    • PHP版本要求: > v5.1
    • 基本函数
      1. spl_autoload(string $classname [, string $fileextensions ]) : __autoload函数的默认实现, 它会去include_path中寻找$class_name(.php/.inc)
      1. spl_autoload_call(string $class_name) : 尝试手动调用所有已注册__autoload()函数来装载请求类;
      1. spl_autoload_extensions([ string $file_extensions ]) : 注册并返回spl_autoload()函数使用的默认文件扩展名
      1. spl_autoload_functions() : 返回所有已注册的__autoload()函数的数组, 如果自动装载函数队列未激活，则返回FALSE。如果没有已注册的函数，则返回一个空数组。
      1. spl_autoload_register([ callable $autoload_function [, bool $throw = true [, bool $prepend = false ]]]) : 将函数注册到SPL __autoload函数队列中。如果该队列中的函数尚未激活，则激活它们
      1. spl_autoload_unregister() : 注销已注册的__autoload(mixed $autoload_function )函数
